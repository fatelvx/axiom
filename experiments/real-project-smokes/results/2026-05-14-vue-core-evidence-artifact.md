# Vue Core Evidence Artifact Calibration

Date: 2026-05-14

Repository: `https://github.com/vuejs/core.git`

Refs: `v3.4.0` -> `v3.5.0`

This run checks whether the Evidence Artifact Loop can make a framework/compiler/runtime workspace reviewable across versions:

```text
.axi contract + scan scope + graph baseline + review story + visible intentional debt
```

It is a calibration artifact, not a verdict about Vue's architecture or maintainer intent. The contract was inferred from `v3.4.0` and reused externally against `v3.5.0`.

## Safety Posture

- Clone-only shallow Git checkouts.
- No target repository installs.
- No target package-manager lifecycle scripts.
- No target builds, tests, `npx`, submodules, or GitHub Actions.
- Axiom used its existing local dependency tree and built CLI.

## Artifact Loop

- Contract: temporary `.axi` inferred from `v3.4.0` with `axi infer --group-by workspace`.
- Scan scope: `--include "packages/*/src/**"`.
- Exclusions: tests and specs under `packages/*/src/**`.
- Graph baseline: unfiltered `axi graph --json` from `v3.4.0`, used as the drift baseline for `v3.5.0`.
- Review story: generated by `axi observe --baseline --markdown` on the current ref.
- Visible intentional debt: none in baseline or current scan.

Drift:

Main hard signal:

- `Shared -> Reactivity` via `packages/shared/src/toDisplayString.ts:2`, importing `@vue/reactivity`.
- In the inferred baseline contract, `Shared` did not declare `Reactivity`; the current ref therefore reports `undeclared_dependency`.

Main advisory signals:

- `Shared` is the strongest graph center: fan-in from 11 modules and fan-out to `Reactivity`.
- Compiler/runtime modules remain visible hubs: `RuntimeCore`, `CompilerCore`, `CompilerDom`, `CompilerSfc`, `ServerRenderer`, and `Compat`.
- One deep internal import remains visible: `Compat -> RuntimeCore` through `packages/vue-compat/src/index.ts:18`.
- No unresolved import warnings appeared in this scoped run.

Removed edges came mostly from playground, template explorer, and declaration-test source groups. This is useful because the artifact shows both new drift and removed drift without treating either as automatically good or bad.

## Review Story Signal

The generated review story put the hard contract failure first, then pointed reviewers to coupling concentration around `Compat` and compiler modules. That is the right order for an evidence artifact:

- stable facts first: exact edge, file, line, and violated declared dependency
- advisory pressure second: graph centers and deep import hints
- caveat always present: static imports are review evidence, not semantic architecture proof

The raw observe and diff Markdown contained a temporary local baseline path, so this public report keeps the sanitized summary and machine JSON instead of publishing raw local-path output.

## Calibration Classification

- Repo shape: framework/compiler/runtime TypeScript workspace with package-level source groups and internal package imports.
- Safety posture: clone-only, no target installs, no target scripts, no submodules.
- Scope question: can a baseline contract plus graph baseline make minor-version source graph drift reviewable?
- Axiom command surface: `axi infer --group-by workspace`, `axi graph --json`, `axi diff`, `axi observe --baseline`, `--include`, `--exclude`, `--warn-coupling-concentration`, `--warn-deep-internal-imports`, `--warn-unresolved-imports`.
- Main signal: one precise new baseline-spec violation, seven removed edges, eight advisory warnings, and zero unresolved warnings.
- Gap class: `quiet-control` for resolver completeness in this shape, plus `advisory-signal-calibration` for coupling and deep-import interpretation.
- Decision: do not change validator behavior from this run; keep it as evidence that the artifact loop can make version drift reviewable without target-specific tuning.
- Code changed: no.
- Follow-up: add another missing portfolio shape, preferably a non-pnpm workspace, UI component library, generated-code-heavy repo, or framework app.
